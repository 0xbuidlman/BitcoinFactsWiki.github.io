Abstract

Transactions could be treated as non atomic entities.

Influences

This page proposes a command line interface based to transactions based on the "Object" lifecycle from ObjectOriented programming. It also borrows from RESTful architecture design (get, put, update, delete, resources via URI). The "Functional design" influences are inspired by Da2ce7:New Transaction and Erlang.
